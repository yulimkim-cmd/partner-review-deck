# Partner Review Deck

파트너 대면 미팅용 실적 리뷰 장표를 python-pptx로 자동 생성하는 스킬.

## 장표 구조 (8슬라이드)

1. **커버** — 좌우 2패널 디자인 (브랜드 컬러)
2. **상반기 전체 성과** — 성장/하락 분해 + 월별 YoY 테이블 (송객/유입/전환율)
3. **노선별 실적** — 송객순 정렬 + 주요 상품 하이라이트 + YoY
4. **비수기 프로모션** — 유입 MoM 하락 데이터 + 프로모션 효과 + 투자 관점 제안
5. **딥다이브** — 파트너별 핵심 이슈 (고객 여정 분석 / 역성장 원인 등)
6. **우수파트너 효과** — 선정 전후 비교 + KPI 충족 현황
7. **하반기 협업 방향** — 파트너 공유용 (내부 아젠다 제외)
8. **Thank You**

## 슬라이드 작성 원칙

### 1. Action Title (제목 = 결론)

모든 슬라이드 제목은 메시지 형태로.

```
✗ "매출 현황"
✓ "송객 +60.8% 성장. 사파리 등 신규 4개 상품이 9,109명 창출"
```

- 5초 안에 핵심 파악 가능
- 숫자 포함 필수
- 1슬라이드 1메시지

### 2. 피라미드 구조

결론 먼저, 근거는 뒤에.

### 3. So What 해석

차트/데이터에 반드시 해석 포함. 데이터만 나열하지 않음.

### 4. 파트너 대상 언어 (필수)

파트너에게 공유하는 장표이므로 플랫폼 내부 용어 사용 금지.

| 플랫폼 용어 | 파트너 용어 |
|------------|-----------|
| UV | 유입 / 유입량 / 방문자 |
| CVR | 전환율 |
| CTR | 클릭률 / 노출→방문 비율 |
| imp | 검색 노출 |
| 퍼널 | 고객 여정 |
| MoM | 전월비 |
| Source: 테이블명 | 내부 데이터 (출처 간소화) |

### 5. 제외 항목

파트너 장표에 넣지 않을 것:

- GMV / 거래액 / 마진 / 수수료 / CM
- 도시 TNA 내 순위 (#2, #12 등)
- 내부 아젠다 / 락인 화법 / 내부 전략
- Source 테이블명 (MART_SALE_D 등)
- "실패" 등 부정적 단어 → 객관적 데이터로 대체

## 디자인 시스템

### 컬러

```python
MRT_BLUE      = '#00AFEC'  # 프라이머리
MRT_BLUE_DARK = '#007BA3'  # 헤더 배경
MRT_BLUE_SOFT = '#E5F7FD'  # 팁바 배경
NAVY_DARK     = '#0D1B2A'  # 커버/엔딩 배경
RED           = '#C0392B'  # 액센트/경고
GREEN         = '#27AE60'  # 긍정/성장
WHITE         = '#FFFFFF'
DARK          = '#1A202C'  # 본문 텍스트
GRAY          = '#718096'  # 보조 텍스트
LIGHT_GRAY    = '#F7FAFC'  # 테이블 교차 배경
BORDER        = '#CBD5E0'  # 테두리
```

### 폰트

Pretendard (한국어 기본)

### 헤더

MRT_BLUE_DARK 바(높이 0.95") + RED 액센트 라인(4pt)

### KPI 카드

WHITE 배경 + BORDER 테두리 + 상단 MRT_BLUE 바(4pt)

### 테이블

- 헤더: MRT_BLUE_DARK 배경 + WHITE 텍스트
- 행: WHITE / LIGHT_GRAY 교차
- 합계행: MRT_BLUE_SOFT 배경 + 볼드

### 팁바

MRT_BLUE_SOFT 배경 + 좌측 MRT_BLUE 바(5pt) + ◆ 아이콘

### 하이라이트 테이블

주요 상품 행: MRT_BLUE_SOFT 배경 + 볼드 + MRT_BLUE_DARK 텍스트

## 워크플로우

1. 파트너 ID 확인
2. 데이터 추출 (월별/상품별 송객, UV, CVR, 확정률)
3. YoY 비교 (전년 동기 데이터 함께 추출)
4. 성장/하락 분해 분석
5. 우수파트너 선정 이력 확인 (있으면 전후 비교)
6. python-pptx로 PPTX 생성
7. stop-slop 점검 (AI 특유의 뻔한 표현 제거)
8. 파트너 대상 언어 교정
9. 내부 회의록은 별도 문서로 분리 (두괄식: 아젠다→장표링크→보조지표→팔로업)

